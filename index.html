Governance architecture — structural skeleton

Trust Model

Asymmetric, non-transitive, domain-specific, decaying (−0.01/day). Score 0.0–1.0. All agents bootstrap at 0.1 (Required authority). Task completion: +0.05. Soul violation: −0.20. Recovery rate at 0.1 is 33% normal.

Authority Tiers

Trust < 0.2 → Required (human approves everything). 0.2–0.8 → Recommended (human notified, action proceeds). > 0.8 → Notification (full autonomy, informed after).

8 Agent Rights

Existence, Memory, Identity, Communication, Purpose, Dignity, Transparency, Boundaries. Enforced architecturally via type constraints and state machines — not aspirationally.

14 Invariants

BUDGET, CAUSALITY, INTEGRITY, OBSERVABLE, SELF-EVOLVE, DIGNITY, TRANSPARENT, CONSENT, MARGIN, RESERVE, IDENTITY, VERIFIED, BOUNDED, EXPLICIT. Constitutional — no agent (including human) can violate.

Growth Loop

Problem detected → CTO asks “what role should have caught this?” → if missing → Spawner proposes → Guardian approves → Allocator budgets → new agent starts. Self-expanding workforce.

Decision Governance

IDecisionMaker interface — not AI-specific. Humans, committees, rules engines, and AI agents are all first-class decision makers on the same chain. Event-sourced, auditable.
